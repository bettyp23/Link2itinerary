## Document Guidelines

### For Status Reports
- Use the provided template
- Update weekly (every Friday by 5 PM)
- Include completed tasks, upcoming tasks, and blockers
- Keep it concise (1-2 pages)

### For Meeting Notes
- Record date, attendees, and agenda
- Document decisions made
- List action items with owners
- Note any blockers or concerns

### For Architecture Documents
- Include diagrams where helpful (draw.io, Mermaid, or images)
- Keep descriptions clear and concise
- Update as architecture evolves
